PM is the memory layer your Claude Code sessions have been missing. It tracks every epic in flight, maintains an explicit detour stack so interruptions never lose their place, and enforces a reconcile gate on resume — all without replacing the tools you already use. Install it once, run `/pm:init`, and it stays oriented across every context compaction.

## What PM Solves

Every Claude Code session that spans more than one context window faces the same three questions the moment the context is compacted or an interrupt hits:

1. What were we working on before the detour?
2. What work is currently outstanding?
3. What is the next highest-priority item?

PM answers all three — mechanically, not from memory.

## Key Features

<CardGroup cols={3}>
  <Card title="Detour Stack" icon="layer-group">
    PUSH/POP your working context around any interruption. The stack survives compaction.
  </Card>
  <Card title="Reconcile Gate" icon="shield-check">
    A mandatory fresh-context review fires on every resume — before you write a single line of code.
  </Card>
  <Card title="Zero Dependencies" icon="feather">

  </Card>

  </Card>
